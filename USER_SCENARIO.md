# 사용자 시나리오 (User Scenarios)

## 페르소나 (Persona)
**이름:** 김철수 (30대, 임베디드 하드웨어 엔지니어)
**배경:** 다양한 신규 칩셋을 사용하여 회로를 설계해야 함. 데이터시트를 일일이 정독하는 데 많은 시간을 소비하고 있으며, 스펙을 놓쳐서 설계 실수를 하는 것을 두려워함.
**목표:** 데이터시트의 핵심 스펙을 빠르게 파악하고, 회로 설계 시 주의해야 할 체크포인트(핀 연결, 전원 등)를 자동으로 추출하여 EDA 툴(CubicLDRC)과 연동하고 싶음.

---

## 시나리오 1: 초기 설정 및 준비
1.  **앱 실행:** 철수는 `Datasheet Analyzer` 애플리케이션을 실행합니다.
2.  **API 키 설정:** OpenAI API를 사용하기 위해 [API 설정] 버튼을 누릅니다. 개인의 OpenAI API Key(`sk-...`)를 입력하고 저장합니다.
3.  **작업 폴더 지정:** 데이터시트 파일들이 저장될 로컬 폴더(예: `C:\Project\Datasheets`)를 [폴더 선택] 버튼을 통해 지정합니다.
4.  **연동 툴 경로 확인:** 자동으로 PCB 설계 검증 툴인 `CubicLDRC`의 경로가 설정되어 있는지 확인합니다.

## 시나리오 2: 데이터시트 자동 분석
1.  **파일 복사:** 철수는 신규 부품 `MAX5033.pdf`를 다운로드하여 위에서 설정한 작업 폴더에 복사해 넣습니다.
2.  **자동 감지:** 애플리케이션이 파일 생성을 자동으로 감지하고, 목록에 `MAX5033.pdf`를 추가하며 상태를 **"Processing"**으로 변경합니다.
3.  **AI 분석:** 백그라운드에서 PDF를 이미지로 변환하여 GPT-4 Vision 모델에 전송합니다. 모델은 데이터시트를 읽고 벤더 코드, 핀 맵, 주요 전기적 특성, 권장 회로 등을 분석합니다.
4.  **완료 알림:** 분석이 끝나면 상태가 **"Finish"**로 바뀌고 결과가 로컬 DB에 저장됩니다.

## 시나리오 3: 분석 결과 검토 및 설계 적용
1.  **결과 확인:** 철수는 목록에서 `MAX5033.pdf`를 클릭합니다.
2.  **부품 정보 탭:** '부품 정보' 탭에서 입력 전압 범위(7.5V~76V), 출력 전류(500mA) 등 핵심 파라미터가 정리된 표를 봅니다.
3.  **분석 결과 탭:** 상세한 기능 설명과 권장 커패시터 값, 저항 분압 계산 예시 등을 읽어봅니다.
4.  **체크포인트 확인:** 우측 하단의 '체크포인트' 목록을 봅니다.
    *   *"VIN 핀에 47µF 전해 콘덴서와 0.1µF 세라믹 콘덴서를 병렬 연결했는가?"* 와 같은 항목을 확인합니다.
5.  **코드 연동:** 체크포인트를 선택하면, 해당 체크 항목을 검증할 수 있는 `Python 코드`가 생성되어 표시됩니다. 철수는 이 코드를 복사하여 자신의 설계 검증 스크립트에 포함시킵니다.

## 시나리오 4: 재분석 (설계 변경 시)
1.  **파일 업데이트:** 데이터시트의 개정판이 나와 파일을 덮어쓰거나, 분석 결과가 미흡하다고 판단될 경우 파일을 다시 체크합니다.
2.  **재분석 요청:** [재분석] 버튼을 클릭하여 기존 DB 기록을 지우고 다시 AI에게 분석을 요청합니다.
